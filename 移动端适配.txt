像素：计算机屏幕显示的最小区域
  设备像素：手机设备的物理像素，比如手机多少像素的分辨率
  CSS像素：web浏览器中布局的像素

当设置了css像素时候，因为考虑到浏览器的缩放，css像素不一定等于设备像素
在苹果iphone4-6视网膜屏中，因为像素密度是普通屏幕的2倍，所以1个css像素=2个设备像素，
dpr(设备像素比) = 设备像素 / 理想视口像素
通过设置<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />
这句话的意思就是
width = device-width 布局窗口 = 理想窗口
initial-scale: 1.0 页面初始缩放为1.0，就是不对css像素进行缩放 默认缩放率为设备像素 / 理想视口像素
user-scalable: no 禁止用户手动缩放
当设备分辨率为750的时候，设备像素比 = 设备像素 / 理想视口像素 750 / 375
在缩放比例为1的时候，1个css像素 = 2个设备像素
缩放比例为设备像素比的倒数

在设计稿为750像素下，布局视口就是375，此时设备像素比是2,1个css像素=2个设备像素
可以显示为在375屏幕下得显示750的页面，所以必须得是缩小1 / window.devicePixelRatio

做移动端适配
1、让布局视口等于理想视口
让页面元素跟着设备变化而等比缩放

步骤：
第一种做法：
  1、让布局视口等于理想市口 width=device-width
  2、因为设备分辨率的原因，设备像素越大，显示的元素就越小，所以必须必须让1css像素等于1个设备像素
  3、支持相等的设备像素比而布局视口不同的页面等比例缩放，所有元素的盒子和定位单位都基于rem，设置页面的rem = (window.screen.width * window.devicePixelRatio) / 10;
  让rem根据不同的布局视口做自动缩放

第二种做法：
  1、让布局视口等于理想视口 width = device-width
  2、将设计稿尺寸当做布局视口尺寸，将1rem = 100px，得到设计稿尺寸下的布局视口的rem，在算出在不同的布局视口，1rem等于多少px
